Angular2.x指令引入问题

发表于 2016-11-16   |   分类于 前端   |  

如果你是Angular1.x的开发者，相信你也会在Angular2.x的指令引入上遇到麻烦。Angular1.x的指令的作用域是全局，只要在顶层模块引入了指令模块，全应用都可以使用，而到了Angular2.x版本，首先你必须在Module的declarations中声明该指令，并且在exports中导出这些指令，然后在需要用到该指令的地方再引入该模块!

spring-boot 打成war包

发表于 2016-10-20   |   分类于 后端   |  

1.修改配置文件pom.xml

2.修改依赖 将scope修改成provided,这样tomcat的包就不会打入war

3.修改启动类继承SpringBootServletInitializer类，并重写SpringApplicationBuilder方法

4.mvn clean package

5.请在tomcat7.0及以上部署
